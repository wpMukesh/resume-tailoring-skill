 Project - DrawInSync

Project description:
DrawInSync is a real-time collaborative whiteboard prototype. It lets authenticated users create or access boards and draw shared shapes on a canvas while a standalone WebSocket server broadcasts drawing events to users connected to the same board.

Stack:
TypeScript, Node.js, Express.js, WebSocket/ws, Next.js 15, React 19, NextAuth Credentials, Prisma, PostgreSQL, JWT, Argon2, Zod, pnpm workspace, Turborepo, Tailwind CSS.

Architecture:
- Monorepo with separate apps for frontend, HTTP server, and WebSocket server.
- Shared packages for Prisma database access, validation/types, TypeScript config, Tailwind config, UI, and ESLint config.
- Express HTTP API handles auth, board management, collaborator access, board lookup, and WebSocket token generation.
- WebSocket server authorizes connections with JWT and stores board-scoped connection buckets for broadcasting shape messages.
- Prisma schema models users, boards, collaborators, and drawable elements with unique constraints, composite keys, indexes, timestamps, and soft-deletes.

Core workflows:
- User signup/signin with Zod validation, Argon2 password hashing, and JWT issuance.
- NextAuth Credentials login calls the custom backend signin API and stores backend JWT data in the session.
- Board creation with owner automatically added as an editor collaborator.
- Owned/shared board listing through Prisma relation filters.
- Collaborator upsert by username and board ID.
- Slug-to-board lookup, board detail lookup, access checks, element lookup, and short-lived WebSocket token generation.
- Canvas drawing for rectangle, circle, and line tools with pending/synced state on the client.
- Per-board WebSocket broadcasting of shape events.

Resume bullets:
- Built a TypeScript monorepo for a collaborative whiteboard with separate Next.js frontend, Express HTTP API, WebSocket service, shared Prisma client, and shared validation packages.
- Implemented authentication APIs using Zod validation, Argon2 password hashing, JWT issuance, and Prisma-backed user lookup/creation.
- Designed a PostgreSQL/Prisma schema for users, boards, collaborators, and drawable elements, including role enums, unique board slugs, composite collaborator keys, timestamps, soft deletes, and board indexes.
- Built protected board APIs for owned/shared board listing, board creation, collaborator role upsert, slug resolution, board access checks, element retrieval, and WebSocket token generation.
- Integrated NextAuth Credentials with a custom backend signin endpoint and propagated backend JWTs into server-side authenticated frontend requests.
- Implemented a canvas drawing engine that supports rectangles, circles, and lines, tracks pending vs synced shapes, translates screen coordinates into virtual canvas coordinates, and redraws canvas state on updates.
- Prototyped real-time board synchronization using a standalone WebSocket server with JWT-authorized connections and board-scoped broadcast groups.

What this project proves:
- Comfortable building across backend, frontend, database, authentication, and real-time layers.
- Understands separation of services: HTTP API vs WebSocket server vs frontend vs shared packages.
- Can model relational backend domains with Prisma and use JWT-based access for both REST and WebSocket flows.
- Has hands-on experience with canvas and real-time event propagation.

Use careful wording:
- Good wording: "prototyped real-time synchronization", "board-scoped WebSocket broadcasting", "collaborative whiteboard prototype".
- Avoid overclaiming: do not say it is production-scale, Redis-backed, fully persistent, conflict-free, or horizontally scalable.

- Tweet and YouTube preview handling while adding content.

- Admin signup/signin.
- Admin course creation, update, and owned-course listing.
- Public course listing.
- User course purchase and purchased-course listing.

Resume bullets:

- Added credential validation and secure password handling using Zod schemas, bcrypt hashing, bcrypt comparison, and duplicate-email handling.

- Built course listing and purchased-course APIs that join course and creator details from MongoDB documents.
